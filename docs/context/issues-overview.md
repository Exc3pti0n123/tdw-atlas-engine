# Issue Overview

Use this file for large prepared workstreams, planning handoffs, or issue clusters that benefit from a single entry point.

## Usage

1. Link the relevant epic or main issue
2. Summarize the prepared issue split
3. Note any ordering or dependency expectations
4. Link supporting requirements/specification docs when applicable

## Rule

Prepared epics should not begin implementation from issue titles alone.
They should point to explicit requirements and specification docs first.
