# Architecture Decision Records (ADR)

ADRs capture architecture decisions separately from changelog and commit messages.

## Scope Rule

From v0.1.4 onward, architecture-affecting changes require:

1. a new ADR, or
2. an explicit "No ADR required" statement in the merge capsule.

## Naming

- `ADR-XXXX-short-title.md`
- Example: `ADR-0001-adapter-factory-runtime-selection.md`

## Status Lifecycle

- Proposed
- Accepted
- Superseded
- Deprecated

## Template

Use:
- `template.md`

## Current ADR Index

1. `ADR-0001-docs-governance-v0.1.4.md`

## Note on Docs-Only Changes

Docs-only work can still require an ADR if it changes architecture/process governance rules.
